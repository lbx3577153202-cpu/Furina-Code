# ADR-0001: Python Baseline and Dependency Policy

**状态:** 已采纳并生效
**生效提交:** `c0565f1f9d40521a96e2e873faf8972c91d192a1`
**生效来源:** PR #2 合并
**日期:** 2026-07-11

---

## 决策

- `requires-python >=3.12`
- 本地开发基线 Python 3.13.x
- CI 验证 Python 3.12 和 3.13
- 零第三方运行时依赖
- pytest 是唯一当前开发依赖
- setuptools 仅为构建依赖
- src-layout
- Git 通过 subprocess 参数列表调用
- Pydantic 和 Agent 框架当前不引入

---

## 为什么不继续使用 3.11

Python 3.11 在能力上可以运行当前基础代码（`tomllib` 从 3.11 开始进入标准库），但不属于本项目冻结的最低兼容线。E1 已冻结最低兼容为 `>=3.12`，本 ADR 遵循该决策。

## 为什么不强制本地安装已经进入 security-only 的 3.12

Python 3.12 当前处于 security-only 阶段（仅修复安全问题，不引入新特性）。本地开发应使用最新的正式 bugfix 版本（3.13）以获得最佳性能和工具支持。CI 仍验证 3.12 以确保兼容性。

## 为什么最低兼容仍保留 3.12

Python 3.12 当前处于 security-only 阶段，但许多用户可能仍在使用。保留 3.12 作为最低兼容可以扩大用户基础，同时 3.13 作为推荐版本。

## 为什么不立即使用 Python 3.14 作为唯一基线

Python 3.14 当前为正式 bugfix 版本，并非不稳定或不可用。本轮继续使用 3.13 作为本地开发基线，原因：
- 本机已经安装并验证 3.13.14
- E1 已冻结最低兼容线为 3.12
- E2 先控制验证矩阵规模
- 后续是否加入 3.14 应由单独兼容性评估决定
