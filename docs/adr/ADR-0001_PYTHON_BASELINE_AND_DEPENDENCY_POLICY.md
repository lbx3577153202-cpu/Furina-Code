# ADR-0001: Python Baseline and Dependency Policy

**状态:** 已冻结
**日期:** 2026-07-11

---

## 决策

- `requires-python >=3.12`
- 本地开发基线 Python 3.13.x
- CI 验证 Python 3.12 和 3.13
- 零第三方运行时依赖
- pytest 是唯一当前开发依赖
- setuptools 仅为构建依赖
- src-layout
- Git 通过 subprocess 参数列表调用
- Pydantic 和 Agent 框架当前不引入

---

## 为什么不继续使用 3.11

Python 3.11 不支持 `tomllib`（3.11 引入但 3.12 才稳定），且 3.12 引入了更好的错误消息和性能改进。3.12 是当前最老的受支持版本（截至 2026-07），适合作为最低兼容线。

## 为什么不强制本地安装已经进入 security-only 的 3.12

Python 3.12 已进入 security-only 阶段（仅修复安全问题，不引入新特性）。本地开发应使用最新的稳定版本（3.13）以获得最佳性能和工具支持。CI 仍验证 3.12 以确保兼容性。

## 为什么最低兼容仍保留 3.12

3.12 是最后一个有完整社区支持的版本，且许多用户可能仍在使用。保留 3.12 作为最低兼容可以扩大用户基础，同时 3.13 作为推荐版本。

## 为什么不立即使用 Python 3.14 作为唯一基线

Python 3.14 尚未发布稳定版（截至 2026-07）。使用 3.13 作为本地开发基线，3.12 作为最低兼容，可以在稳定性和先进性之间取得平衡。
