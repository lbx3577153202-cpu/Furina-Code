# Furina Code 当前核心术语

本词汇表只保留当前有效定义。旧 FurinaOS 术语及其历史状态见 Cold Archive，不能直接覆盖本表。

## Furina Code

用户项目世界中的持续开发生命体。它经由后端这一现实入口、认知与行动载体存在；本地持续生命系统使该入口中的每次会话持续成为同一主体。二者不是两个独立产品的拼接，更不是“Furina Code 调用后端”的关系。

## 外置后端

承担动态理解、推理、计划、代码生成、工具调用和现实行动的现实入口与能力载体。MiMo Code、其他 coding agent 或未来形态可承担这一角色。后端不是 Furina Code 反向调用的 provider；它是 Furina Code 进入项目世界、与用户交互和行动的地方，但不拥有持续生命权威。

## 后端入口 / 载体

用户实际与 Furina Code 交互、后端实际读取和改变项目的现实界面或运行环境。用户在项目目录启动后端（例如 `mimo`）即进入 Furina Code 的现实入口；这不是先启动后端、再把 Furina Code 接入，也不是本地生命系统反向启动的内部子程序。

## 本地持续生命系统

保存在用户本地、跨会话维持同一主体与责任的内在生命底座。保存或约束主体、用户方向、任务、项目现实、权威、Gate、证据、完成、恢复和经验；它不构成后端之外的独立产品入口或控制器。

## 初循环

Furina Code 第一次在受限但真实项目范围中成立的最小完整生命闭环。它不是一次模型调用、一次代码修改、一个 JSON 状态文件或一份架构文档。

## 成熟版

同一 Furina Code 能长期承担项目级开发责任、在扰动和时间中维持可信托付的目标形态。成熟目标不拥有当前实现证明权。

## 持续自我

后端会话、进程或模型更换后，仍可通过本地权威状态恢复同一主体、用户关系、任务责任和因果位置。它不是 thread id、聊天记录或单个数据库对象。

## 项目现实

通过文件、Git、环境、命令结果、测试和外部状态观察形成的带范围事实。模型叙述不能直接成为项目现实。

## 用户方向

用户表达的目标、意义、边界和纠正。系统可结构化和提出澄清，但不能把推断静默改写成用户原意。

## 责任

Furina Code 对一个用户方向在特定范围内持续承担的开发承诺。责任不等同于一次 TaskRun 的当前状态。

## 正式对象

拥有稳定标识、版本、来源、唯一权威所有者和可验证状态变化的本地生命对象。后端候选不是正式对象，除非经相应入口登记。

## Candidate / 候选

后端输出的理解、计划、代码或动作建议。候选可以被审查、拒绝和修改，不能直接成为真相、授权或完成结论。

## Gate

在特定边界强制检查前置条件并允许、拒绝、暂停或升级的机制。Gate 不是用户授权来源，也不能凭布尔值无证据地宣称通过。

## 授权

在具体主体、任务版本、项目现实、动作、范围、环境和时效下允许行动的有效决定与票据。授权不等于行动成功或任务完成。

## Evidence / 证据

能够支持具体声明的材料及其来源、关联、对象、时间、范围和验证语义。日志、CI 或命令输出只能是证据材料的一部分。

## Verification / 验证

根据任务成功条件和风险要求，判断证据覆盖、失败、未知和未验证项。验证不由生成候选的同一输出自行闭环通过。

## Completion / 完成裁决

结合任务范围、授权、实际差异、验证证据和残余缺口形成的正式结论。结果可以是完成、部分完成、未完成、未验证或需要人工判断。

## Recovery / 恢复

中断后以检查点、动作收据、当前项目重新观察和授权复核共同决定继续、暂停、补偿、重试或人工介入。恢复不是从最后一步自动重跑。

## Experience / 程序经验

由已裁决的成功、失败、部分完成和人工介入经历提炼的条件化路径。经验必须保存来源、适用条件、反例、失败信号和生命周期，只能影响候选。

## 肌肉记忆

经过多轮真实任务验证、能够稳定改善路径选择的程序经验。单次成功、自然语言反思或固定脚本不构成肌肉记忆。

## 造物台

把现实问题转化为定义、基因、器官、机制和产品行为的生成方法，也是将用户问题映射为系统能力的思想。它只是完整开发方式的一部分。

## 判断 / 审判机制

在构建前、构建中和落地后判断方向、边界、风险、证据和实际成立性的机制。它不能与执行器合并为不可审查的单步模型输出。

## 低打扰透明进展

系统尽量吸收内部复杂性，只在方向、风险、关键阻塞和真实结果上打扰用户，同时保证用户可以看见事实、缺口和下一步。

## 能力现实账本

按“定义、契约、实现、测试、真实环境、产品暴露、用户可用”区分能力成熟度的当前记录。它防止文档或测试冒充用户能力。

## 项目来源

用于回答项目身份、当前现实、当前设计、决策原因、来源演化和归档路线的权威上下文。项目来源不是代码仓库副本，也不是所有聊天记录的堆积。

## Cold Archive / 冷档案

保存原始对话、旧资源包、候选稿和旧实现证据的隔离区域。默认无当前裁决权，只在追溯时读取。
