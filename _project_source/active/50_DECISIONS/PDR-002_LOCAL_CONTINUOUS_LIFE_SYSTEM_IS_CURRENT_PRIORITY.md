# PDR-002：本地持续生命系统是当前工程重点

## 状态

`ACCEPTED`

## 背景

现实中的 coding backend 已经能够理解、推理、读取仓库、修改代码和运行测试。当前项目最缺的不是再次制造基础智能，而是让这些能力成为同一个、持续、诚实、受用户主权约束的 Furina Code。

## 决定

当前工程优先建设：

- 持续主体和用户/项目绑定；
- 任务和责任跨会话存在；
- 项目现实观察和版本绑定；
- 授权、Gate、动作收据和对账；
- 证据、验证和完成裁决；
- 恢复；
- 经过真实反馈的经验成长；
- 后端入口与这些状态的耦合。

不把下列事项作为当前优先目标：

- 从零训练或重造模型；
- 为多个后端提前写大量 Adapter；
- 建设成熟九系统全部基础设施；
- 因“以后可能用”添加无人值守脚本；
- 用通用框架替代生命权威设计。

## 后果

每个新任务必须回答：

1. 它补的是哪条生命因果链；
2. 它给用户带来什么真实变化；
3. 为什么不能直接使用现有后端完成；
4. 它是否增加了本地持续权威；
5. 它如何被验证并进入现实账本。
