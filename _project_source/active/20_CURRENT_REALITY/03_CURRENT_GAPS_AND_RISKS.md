# Furina Code 当前缺口与风险

## P0：产品根关系仍未落地

### R-001 后端入口与本地生命系统尚未正式耦合

当前有工程闭环，但没有 MiMo Code 或其他后端在产品入口中加载、读写并服从本地生命状态的正式协议与实现。

### R-002 产品入口缺失

仓库有内部受控闭环和测试，没有用户可持续交互的 Furina Code 产品入口。

## P1：初循环扩展风险

### R-003 受控写入范围仍极窄

已建立 `notes/*.txt` 单文件创建的闭环；未覆盖更新、删除、重命名、多文件依赖、跨仓库、外部系统或高风险动作。不得将首个切片外推为“已可自主改代码”。

### R-004 通用恢复裁决仍缺失

本轮只证明单文件写入的中断/未知结果恢复：绝不盲重试；复杂写入和外部副作用的安全恢复仍未证明。

### R-005 可复用经验成长仍缺失

已证明候选经验的第二轮受限试用；尚未获得 `reusable`，也没有长期、多任务或跨任务族经验。

### R-006 初循环结论不能外推

IL-L3 仅在隔离 Git 项目、低风险 `notes/*.txt` 单文件创建任务族内成立。每个新增任务族都必须新增真实任务、Gate、对账、失败反例和范围化建立记录。

## P1：来源治理风险

### R-007 理论基因 V1 原文缺失

不得根据摘要伪造冻结原文。

### R-008 历史来源携带过时现实

旧 HEAD、旧测试数和“初循环未成立”的历史快照不得覆盖本包当前事实。

### R-009 依赖方向误读

不得把本地生命系统写成调用后端的外部控制器，也不得把后端降格为普通 provider。

## P2：工程维护风险

### R-010 本地证据尚未形成新的提交/CI 锚点

405 passed 是本地 Windows 证据窗口；在有新的提交 SHA 和 CI 前，不得宣称 GitHub main 或跨平台 CI 已验证本轮实现。

### R-011 测试数量被当作能力等级

测试数量只证明回归范围，不单独证明入口、用户可用性或广义生命闭环。
